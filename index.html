Análise de Compra

Diagnóstico da causa-raiz: produtos comprados em excesso ou sem giro compatível
O vencido é o sintoma; a compra é a causa. Esta análise cruza o volume parado com o tempo de prateleira do produto para sinalizar compras desproporcionais ao giro. Quando houver dados de venda no sistema, o cruzamento com giro real torna o diagnóstico ainda mais preciso.
Sinais de Compra Excessiva (over-buying)
Lotes únicos de alto valor representam um único pedido grande. Quando o lote inteiro vence, é forte indício de que a quantidade comprada superou a capacidade de venda da(s) loja(s) no prazo de validade.

Recomendações para a Diretoria

Aging — Envelhecimento do Vencido

Distribuição por tempo de vencimento — define urgência e recuperabilidade
Itens vencidos há menos de 30 dias ainda têm maior chance de ação. Atenção à política por linha: Nutrição Animal (FAP/FCB) — sem devolução, destinar como descarte ou venda imediata; Pastagem — devolução somente à FAG (não entre filiais); Agrícola — verificar revalidação de análise; demais — tentar bonificação com fornecedor. Itens há mais de 365 dias são candidatos a baixa/descarte.

Área Restrita — Configurações e Operações

Importação, publicação e sincronização via JSONBin
Operações
Sincronização automática com fallback para o GitHub: a cada importação, o app tenta publicar o arquivo completo (vencidos + a vencer) direto na nuvem (JSONBin) — quando cabe no limite do plano grátis, todo mundo vê na hora. Quando o arquivo é grande demais para a nuvem: no seu navegador (com o Token do GitHub configurado), o app cria sozinho uma branch no repositório com o index.html completo, pronta para você revisar e mesclar. Em qualquer outro navegador (ex.: o da Gabriela, sem token), o app baixa o index.html completo automaticamente para ela enviar para você finalizar a publicação — o Token nunca sai do seu próprio navegador.
Configuração (somente você e a Gabriela)
Guardada apenas no seu navegador. Nunca sai do seu computador.
Só você precisa configurar este token, e somente neste navegador/dispositivo — ele nunca é embutido no arquivo publicado nem enviado para a nuvem, porque dá acesso de escrita ao repositório. Use um fine-grained PAT restrito ao repositório Vencidos-Agroquima, com permissão apenas de Contents: Read and write. Em qualquer outro dispositivo (ex.: o da Gabriela), quando o arquivo for grande demais para a nuvem, o app só baixa o index.html completo para ela te enviar — só você cria a branch, usando este token local.

A senha de operador protege contra bisbilhotagem, não é cofre. Passo a passo — configuração inicial
1 Acesse jsonbin.io e clique em Sign Up (gratuito).
2 Após login, clique no seu nome → API Keys → copie a Master Key.
3 Cole a Master Key ao lado e clique em Criar Bin (ou use o Bin ID já embutido).
4 Clique em Publicar para todos no topo → baixe e suba o index.html no GitHub (uma vez, para embutir o Bin ID).
5 (Só no seu navegador, opcional) No GitHub: Settings → Developer settings → Fine-grained tokens → Generate new token, restrinja ao repositório Vencidos-Agroquima e dê permissão de Contents: Read and write. Cole o token ao lado e clique em Salvar — ele fica só neste navegador, nunca é publicado.
✓ Pronto. Daqui em diante, ao importar um XLSX, todos veem na hora — e se o arquivo for grande demais para a nuvem, no seu navegador (com o token) uma branch é criada sozinha no GitHub para você revisar e mesclar.
